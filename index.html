首先需要一个谷歌账号,任意区不需要英区谷歌账号
我这里用的美区谷歌
1.美区谷歌账号x1
2.任意美卡x1
3.任意的安卓设备（需要root进行mit证书安装）模拟器也行

原理是要插入一串特殊的优惠码 使其强行在订阅的时候显示出来
这些优惠码有
单月0刀试用
3天免费试用
10刀半价优惠
3月免费试用
6月免费试用
12月免费试用
pro挽回试用免费一个月优惠
优惠分两种
一种是插入优惠码 一种是有特定资格的账号（最稳定）

| 3-day-free-trial 3天免费
| plus-1-month-free-trial 1个月免费
| 1-month-10-dollars 10刀优惠
|2wqkodfx51z2x|6个月免费
|2ispxs5mtgz35|12个月免费
pro-winback-1-month-free pro的挽回试用
这些优惠码值只适合hook使用 不适用下面的案例替换

这里是3天，半价，一个月免费试用的优惠码-----优惠码是固定的每次都可以使用，这就是为什么外面的卡网可以一直生产低价plus的原因

安装好mit证书，安卓安装好chatGpt的软件预先格式化的文本之后，我们打开抓包（此方法为ssl paanning）

https://android.chat.openai.com/backend-api/accounts/check/v4-2023-04-27

我们可以找到这么一个接口，这个是chatGpt获取当前登录用户信息的接口
其中在"eligible_promo_campaigns"的参数字段里面，我们可以修改响应值强行塞入一串优惠代码，这边给一个一个月的优惠案例

“eligible_promo_campaigns”: {
“plus”: {
“id”: “plus-1-month-free”,
“metadata”: {
“plan_name”: “chatgptplusplan”,
“title”: “linux.do”,
“summary”: “你已获得 1 个月的 Plus 套餐优惠促销代码。”,
“discount”: {
“percentage”: 100
},
“credit_grant”: null,
“duration”: {
“num_periods”: 1,
“period”: “month”
},
“schedule”: null,
“price_period”: “recurring”,
“promotion_type”: “discount”,
“promotion_type_label”: “1-month free trial”,
“no_auto_renewal_at_discount_end”: true,
“plan_type_change”: null
}
}
},

写好规则替换后，我们重新打开软件点击升级，这时候我们就能看到一个月免费试用的界面了

image411×589 41.4 KB

之后我们就可以点击订阅了（事先给账号绑卡）由于是0刀的原因，谷歌的风控会特别低
只需要过一个验卡环节即可订阅成功

这时候我们只需要拦截这个接口（可以随意重定向一个无效域名，这样就不会使其生效）
https://api.revenuecat.com/v1/receipts
把请求响应的
“fetch_token”: “xxxx”,
token取出，就可以保存我们需要的凭证了

之后我们要给其他账号开的时候，只需要去网页的session里面取出account字段下的id
再加上这个订阅凭证即可给任意账号开通权益，pro同理

目前先说之前的公益bot用的是什么方案来进行大规模开pro账号

ChatGPT的安卓应用是采用第三方的https://api.revenuecat.com/v1/receipts
订阅管理平台，会这个地址来进行校验凭证，谷歌付款显示是单独的
然后前一阵子的无限复制ChatGPT pro就是
需要先正价开一个ChatGPT Pro然后在ChatGPT的软件里面点击降级为plus这时候软件就会向这个接口请求一个新的fetch_token 而这个token就是我们需要用来给其他账号开的订阅凭证，这个凭证每次都是全新的只要没有用过都会生成一个，我们只需要把这个凭证给需要开的账号就可以完成一次Pro的账号订阅

这个是ChatGPT自身的回调漏洞，属于没有与第三方管理好订阅接口导致的

这也是为什么那个公益bot可以无限生产Pro的权益账号，并且需要排队等待的原因，主要还是因为和第三方订阅管理机制没有鉴权好的问题 不存在一个凭证可以无限使用（该方案已修复）预先格式化的文本
